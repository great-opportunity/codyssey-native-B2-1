1. 기획 및 설계

이름: 찐친 (Jjinchin)

  • '찐(진짜) + 친(친구)' = 너의 진짜 한국인 친구
  • 외국인이 배우는 첫 한국어 단어가 곧 앱 이름이 되는 구조

한 줄 포지셔닝

한국인이 진짜 좋아하는 경험을, 그걸 하는 데 필요한 한국어와 함께 안내하는 체험형 한국어 앱.

핵심 가치 제안

  • 외국인 혼자서는 다가가기 어려운 '진짜 한국'을, 현지 친구가 옆에서 알려주듯 안내한다
  • 경험을 준비·이동·현장 단계로 쪼개고, 각 단계에 필요한 한국어를 붙여준다
  • 예약을 하지 않아도, 콘텐츠를 보고 따라 읽는 것만으로 한국과 한국어에 가까워진다

스코프

  • 포함: 경험 큐레이션, 단계별 안내, 상황별 한국어(듣기·따라 읽기·이해), 게임형 진도
  • 제외: 호텔/교통/액티비티 예약·결제·외부 API 연동 → 난이도·비용·차별성 모두에서 우리 강점이 아님

최초 화면 설계

화면 1 — 메인 (홈)

  • 목적: "오늘 뭘 경험하지?"에 즉답 + 학습 동기 유지
  • 상단: 히어로 영상/이미지 (외국인이 골목·시장·찜질방을 즐기는 무드, 안전·즐거움 강조)
  • 중단: 오늘의 찐 경험 추천 카드 1개(크게) → 오늘의 한 마디(한국어 표현 1개, 듣기 버튼) → 카테고리 칩(찐 푸드·동네 산책·로컬 놀이·시장·목욕탕/찜질방·노래방)
  • 하단: 진행 중 학습 진도 바 + 글로벌 네비게이션(홈·탐색·내 학습·프로필)

화면 2 — 목록 (경험 탐색)

  • 목적: 100개 경험을 '내가 혼자 할 수 있는 것'으로 빠르게 추리기
  • 상단: 검색창 + 필터 칩(카테고리 / 난이도(혼자 가능도) / 지역 / 필요한 한국어 수준)
  • 본문: 경험 카드 그리드. 카드 1장 구성 →
  • 썸네일 사진
  • 경험명 (예: 동네 찜질방 즐기기)
  • 난이도 배지 (🟢 혼자 쉬움 / 🟡 보통 / 🔴 도전)
  • 소요시간·지역
  • 배우는 표현 5개 뱃지 (← 차별점 시각화)

화면 3 — 상세 (경험 가이드 + 한국어)

  • 목적: 실제로 해내게 만들기. 차별점이 가장 또렷하게 드러나는 화면
  • 구성(위→아래):
  1. 커버 영상/사진 + 경험명
  2. 공감 스토리: "한국인은 왜 이걸 좋아할까?" 2~3줄
  3. 단계별 안내: ① 준비물 ② 가는 법(이동) ③ 현장에서 할 일
  4. 한국어 표현 카드(각 단계에 인라인): 한글 + 발음(로마자) + 뜻 + 🔊듣기 + 🔍따라 읽기
  5. 현장 미션 체크리스트: 완료 시 뱃지
  6. 저장·공유
